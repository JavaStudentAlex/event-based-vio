# S01: Extrinsics-aware Event+IMU Correction

**Goal:** Modify the event_imu backend to extract and apply IMU-to-camera extrinsics from the MVSEC calibration data, making the event shift to world displacement conversion geometrically correct instead of assuming camera equals body frame
**Demo:** After this: EventImuBackend uses calibrated IMU-to-camera rotation from MVSEC calibration data to transform event-derived displacements into the body frame; synthetic tests prove extrinsics path works correctly and existing tests still pass

## Must-Haves

- `_extrinsics_rotation_from_calibration` returns a Rotation when calibration has `imu_cam_transform`\n- EventImuBackend applies the extrinsics rotation in `_event_world_displacement`\n- Falls back to identity when calibration is absent\n- All existing event_imu tests pass unchanged\n- New test verifies extrinsics-aware correction differs from identity correction when extrinsics are non-trivial

## Proof Level

- This slice proves: contract — synthetic unit tests prove correctness; no live MVSEC run required

## Integration Closure

Upstream: MvsecSequence.calibration, EventImuBackend.run(). New wiring: extrinsics rotation extracted in run() and passed to _event_world_displacement(). Remains: S02 and S03 build on updated backend.
